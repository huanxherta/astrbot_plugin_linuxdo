# 更新日志 (CHANGELOG)

## [1.2.1] - 2026-04-16
### 优化
- **界面极致精简**: 强制使用单一 `Node` 封装所有话题内容，确保在任何指令下均只发送一个合并转发气泡，彻底解决多气泡干扰问题。

## [1.2.0] - 2026-04-16
### 新增
- **最新讨论**: 新增 `/ld_new` 指令，支持获取社区最新的 30 条讨论话题。
### 重构
- **核心逻辑**: 抽象出通用的节点创建工具函数，统一 `/ld_top`、`/ld_new` 和 `/ld` 的展示风格。

## [1.1.5] - 2026-04-16
### 变更
- **取消上限**: 去除 40 条热门话题的硬编码限制，支持获取 API 返回的全部话题。

## [1.1.1] - [1.1.4] - 2026-04-16
### 修复与尝试
- **架构博弈**: 围绕“单气泡展示”与“NTQQ 内核超时 (1200)”进行的多次架构调整，最终确立了单节点全内容方案以保证极致稳定性。

## [1.1.0] - 2026-04-16
### 优化
- **分块打包**: 首次引入分块转发逻辑，解决长文本被系统拒发的问题。

## [1.0.9] - 2026-04-16
### 修复
- **架构对齐**: 按照 AstrBot 官方文档重构合并转发逻辑，弃用 `Forward` 类，直接发送 `Node` 列表。

## [1.0.6] - 2026-04-16
### 修复
- **校验错误**: 修正 Node ID 的数据类型为整数 (Integer)，对齐 Pydantic 校验规范。

## [1.0.2] - 2026-04-16
### 修复
- **绕过屏蔽**: 引入 `curl_cffi` 模拟 Chrome TLS 指纹，完美解决 Cloudflare 403 Forbidden 拦截。

## [1.0.0] - 2026-04-16
### 新增
- **初始版本**: 支持 /ld_top 获取热帖和 /ld 搜索 LINUX DO 社区内容。
