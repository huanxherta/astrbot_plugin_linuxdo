# 更新日志 (CHANGELOG)

## [1.7.0] - 2026-04-16
### 新增
- **大批量抓取支持**: 将默认获取数量提升至 60 条，让信息获取更高效。
### 优化
- **智能分块逻辑**: 引入每 60 条话题自动分 Node 的机制，确保在获取海量数据时不会超过 QQ 的单条消息字数上限。
- **请求效率**: 优化了翻页请求的延迟时间，提升响应速度。

## [1.6.1] - 2026-04-16
### 变更
- **指令更名**: 将 `/ld_fresh` 更名为 `/ld_time`，语义更清晰。
### 优化
- **代码精简**: 移除了冗余逻辑，提升运行效率。

## [1.6.0] - 2026-04-16
### 新增
- **纯净新帖指令**: 新增 `/ld_time` (原 `/ld_fresh`)，专门获取最新创建的话题，彻底排除旧帖回复的干扰。

## [1.5.1] - 2026-04-16
### 修复
- **WebUI 菜单**: 针对 AstrBot v4.22.1 修复了插件配置菜单不显示的问题。

## [1.5.0] - 2026-04-16
### 新增
- **全自动翻页**: 核心逻辑支持多页抓取，打破了 Discourse API 单次 30 条的限制。

## [1.4.0] - 2026-04-16
### 变更
- **稳定性调整**: 移除了不稳定的 `/ld` 搜索功能，规避社区 429 频率限制。

## [1.0.2] - 2026-04-16
### 修复
- **Cloudflare 绕过**: 引入 `curl_cffi` 模拟浏览器指纹，解决 403 Forbidden 问题。

## [1.0.0] - 2026-04-16
### 新增
- **初始版本**: 实现 LINUX DO 热帖获取基础功能。
